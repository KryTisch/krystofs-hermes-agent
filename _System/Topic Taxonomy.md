# Topic Taxonomy

This vault uses a stable, *flexibly extendable* topic taxonomy for personal chats and knowledge capture.

## Core topics
- Lifestyle
- Kochen / Ernährung
- Reisen
- Familie
- Fitness / Gesundheit
- Arbeit / Projekte
- Technik / Tools
- Hermes Agent / Bot Ops
- Obsidian / Wissenssystem
- Meetings / Transkription
- Finanzen / Budget
- Medien / Content
- Social / Kommunikation
- Lernen / Weiterbildung
- Sonstiges / Inbox

## Expansion rule
- New topics are allowed whenever they become recurring.
- If a subject appears only once, keep it in Inbox or as a secondary tag.
- Promote a subject to a core topic only when it is useful often enough to deserve its own thread / folder / tag.
- Prefer small, stable topic names over overly specific ones.
- When several past threads fit one larger theme, consolidate them into that larger theme instead of keeping many tiny lanes.
- Retire narrow subtopics into a broader parent topic when the combined theme is clearly more useful.

## Historical consolidation rule
- Do not only classify the current message.
- Also look back across prior conversations and group related history into durable main topics.
- If older subtopics repeatedly cluster together, they should become one shared main topic.
- Keep a separate thread only if it stays meaningfully distinct in ongoing use.
- Favor long-lived topic buckets over session-specific labels.

## Usage
- Prefer one primary topic per note/chat item.
- Add secondary tags when needed.
- Use consistent tags for cross-linking, e.g.:
  - `#topic/lifestyle`
  - `#topic/kochen`
  - `#topic/reisen`
  - `#topic/familie`
  - `#topic/fitness`
  - `#topic/technik`
  - `#topic/meetings`
  - `#topic/obsidian`
  - `#topic/hermes`
  - `#topic/finance`

## Telegram mapping
- One Telegram thread per core topic when practical.
- Use a hub / index thread for navigation.
- Cross-topic conversations should stay in the dominant topic thread and be linked from related topics.
- If a topic becomes recurring and useful, it deserves its own thread; if not, keep it in the broader parent topic or Inbox.

## Sorting rule
- Put actionable notes in their topic folder or under a topic heading.
- Keep raw incoming material in Inbox until classified.
- When a note spans multiple topics, choose the dominant one and link the rest.
- Reuse the same topic names across Telegram, Obsidian, and other systems.

## Goal
Build a personal "Gedankenpalast" that is:
- searchable
- linked
- stable over time
- easy to extend
- consistent across chat and knowledge base
