# Active Topic Buckets

This note captures the current working set of durable topics inferred from recent conversations and historical usage. It is meant to stay small, stable, and easy to extend.

## Main buckets

### Kratos Coach / Fitness / Gesundheit
Use for:
- training planning
- readiness / Garmin / recovery checks
- workouts and exercise selection
- workout lessons and coaching decisions

### Rezepte / Ernährung
Use for:
- recipe database work
- recipe intake and extraction
- recipe search and categorization
- cookbook / meal planning workflows

### Hermes Agent / Bot Ops
Use for:
- Telegram topics and routing
- gateway ops and crashes
- profiles and cron jobs
- model/provider routing
- bot setup and troubleshooting

### Obsidian / Wissenssystem
Use for:
- vault structure
- topic taxonomy
- note organization
- chat-to-knowledge routing
- long-term knowledge architecture

### Medien / Content / YouTube
Use for:
- transcript extraction
- video analysis
- summaries and content intake
- media-derived knowledge capture

### Meetings / Transkription / Voice
Use for:
- voice-message paraphrase and response handling
- transcription
- meeting notes
- spoken input that should become structured text

### Arbeit / Projekte / Technik
Use for:
- implementation work
- automation
- scripts
- tools
- troubleshooting

### Sonstiges / Inbox
Use for:
- ambiguous material
- one-off items
- things that do not yet recur enough for their own lane

## Consolidation rule
- If two topics keep appearing together and can be summarized under one larger idea, merge them into a single bucket.
- Only split a bucket when the subtopic is recurring enough to justify its own long-lived lane.
- Prefer fewer, stronger buckets over many weak ones.

## Practical use
- One primary bucket per item.
- Secondary links only when useful.
- Mirror these names across Telegram and Obsidian when practical.
